# Mini ERP — Role Dashboards

## Global dashboard rule

- Dashboard is for reading and prioritizing.
- Heavy actions happen through Action Hub or Wizard.
- Each role sees only what it can act on.

---

## ADMIN

## Main goal

- control the operation
- detect incidents
- audit exceptions

### Priority blocks

1. critical alerts
2. cash incidents
3. overdue tasks
4. pending validations
5. business overview

### Recommended cards

- critical cash differences
- mandatory pending tasks
- purchases pending validation
- transfers pending approval
- employees with overdue work

### Quick actions

- audit cash
- validate purchase
- approve transfer
- create task

---

## CAJERO

## Main goal

- operate cash safely
- register movements fast
- close shift with control

### Priority blocks

1. current cash status
2. expected balance
3. today movements
4. closure warnings

### Recommended cards

- cash open/closed state
- opening base
- expected current total
- last movements
- pending closure alerts

### Quick actions

- open cash
- add income
- add expense
- add transfer
- close cash

---

## EMPLEADO

## Main goal

- know what to do now
- see payroll status
- execute tasks with evidence

### Priority blocks

1. pending tasks
2. in progress tasks
3. current payroll
4. remaining payroll amount

### Recommended cards

- task inbox summary
- urgent tasks
- overdue tasks
- current payroll total
- paid vs remaining

### Quick actions

- start task
- continue task
- attach evidence
- view payroll

---

## Task inbox layout

### For employee

- tab `Pendientes`
- tab `En proceso`
- tab `Completadas`

### For admin

- employee list
- each employee opens task view with same logic:
  - pendientes
  - en proceso
  - completadas

---

## What should NOT live in dashboards

- long edit forms
- multi-step financial flows
- full validation process
- task completion without evidence flow
